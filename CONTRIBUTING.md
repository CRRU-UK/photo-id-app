# Contributing

## Prerequisites

- Node.js >= 24 (use `nvm install 24` / `nvm use 24`)
- npm (comes with Node.js)

## Getting Started

1. Fork and clone the repository
2. Install dependencies: `npm ci`
3. Start the app in development mode: `npm start`
4. Run the test suite: `npm test`

## Development Workflow

1. Create a branch from `main` for your change
2. Make your changes
3. Run `npm test` to ensure linting, type checking, and unit tests pass
4. Submit a pull request against `main`

## Pull Requests

Run `npm run changesets:add` (or `npx @changesets/cli add`) before merging pull requests.

## Code Conventions

See `ARCHITECTURE.md` for a comprehensive list of conventions, architecture details, and common pitfalls.

Key points:

- Use `IPC_EVENTS` constants for all IPC communication
- Wrap MobX-consuming React components with `observer()`
- Use the `photo://` protocol for image URLs, not `file://`
- All file I/O happens in the main process (`src/backend/`)
- Use GitHub Primer components and icons for UI consistency
- Run `npm test` before submitting (linting + types + unit tests)

## Testing

- Tests are co-located with source files as `*.test.ts` / `*.test.tsx`
- Mock boundaries (filesystem, Electron APIs), not logic
- Test behaviour, not implementation details

## Architecture

See `ARCHITECTURE.md` for detailed specifications, user journeys, and technical documentation. See `SECURITY.md` for the security model.

## Do Not

- Run `npm run package`, `npm run make`, or `npm run publish`, these are for CI only
- Edit `src/routeTree.gen.ts`, this is auto-generated by TanStack Router
- Remove `console.log` statements, they serve as Sentry production diagnostics
