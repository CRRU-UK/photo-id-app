To report a bug, provide feedback, or submit a feature request, please [open an issue on the GitHub page](https://github.com/CRRU-UK/photo-id-app/issues/new/choose).

## Before reporting

- Check the [troubleshooting page](./troubleshooting.md) for common problems and fixes
- Check the [known issues](https://github.com/CRRU-UK/photo-id-app/issues/355) to see if the problem is already tracked
- Search [existing issues](https://github.com/CRRU-UK/photo-id-app/issues?q=is%3Aissue) for similar reports

## Writing a useful bug report

A good bug report makes it much easier (and faster) for us to fix the problem. The most helpful reports include:

- **What you did** - the steps that led to the bug, in order
- **What you expected to happen**
- **What actually happened** - including any error messages or dialogs
- **Your platform** - Windows, macOS, or Linux, and the OS version
- **The app version** - shown in ==:octicons-question-16: Help > About== from the menu bar
- **Screenshots or a screen recording** if the bug is visual or hard to describe

If telemetry is enabled, error reports are sent to us automatically when something crashes. See the [privacy policy](./privacy.md) for what is and is not captured. Even with telemetry enabled, a written bug report is still useful, as it tells us what you were trying to do, which the automatic report cannot.

## Submitting

When you are ready, use one of the templates on the [issues page](https://github.com/CRRU-UK/photo-id-app/issues/new/choose):

- [Bug report](https://github.com/CRRU-UK/photo-id-app/issues/new?template=bug_report.yaml)
- [Feature request](https://github.com/CRRU-UK/photo-id-app/issues/new?template=feature_request.yaml)
- [Question](https://github.com/CRRU-UK/photo-id-app/issues/new?template=question.yaml)
